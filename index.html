Operations on strings:

  • Concatenation → joins two strings, + is used.
  • Replication → Makes multiple copies of a string using *
  • Comparison → Python follows lexicographic ordering
Lexicographic ordering: The first characters from the two strings are compared. If they differ this determines the outcome of the comparison. If they are equal, then the second character of both the strings are compared. This process continues until either string is exhausted.

How to do anti-aliasing?

Python provides 3 ways to avoid aliasing:

list1 = [1, 2,  3]

# The three ways to avoid aliasing as follow:
list2 = list(list1)
list3 = list1[:]
list4 = list1.copy()

Shallow copy and Deep copy for anti-aliasing:

In lists commonly, if we want to avoid aliasing, for example:

list1 =  [[1, 2, 3], [4, 5, 6]]
list2 = list1.copy()

list1[0][0] = 100

print(list1)
print(list2)

# Outputs:
[[100, 2], [3, 4]]
[[100, 2], [3, 4]]

Solution for the problem is:

from copy import deepcopy
list1 =  [[1, 2, 3], [4, 5, 6]]
list2 = deepcopy(list1)         # Makes a clean copy

list1[0][0] = 100

# Outputs:
[[1, 2], [3, 4]]
[[100, 2], [3, 4]]

Iterating over a dictionary

  • keys()
squares = {0: 0, 1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

    print(f'The square of {key} is {squares[key]}')

The above can also be implemented without keys() as Python makes it even simpler:

squares = {0: 0, 1: 1, 2: 4, 3: 9, 4: 16, 5: 25} 

for key in squares:
    print(f'The square of {key} is {squares[key]}')

key-value pairs can be iterated using items():

squares = {0: 0, 1: 1, 2: 4, 3: 9, 4: 16, 5: 25}    # key is a number, value is its square

for key, value in squares.items():
    print(f'The square of {key} is {value}')

Object Oriented Programming:

A good example to understand would be ‘google forms’. Templates are distributed and each one submits their own form.

Template is similar to class in OOP. Copy of each submission is similar to object

class is a blueprint used to create objects
class Student:
	count = 0
	def __init__(self, name, marks):
    self.name = name
		self.marks = marks
		Student.count += 1

	def print_details(self):
    print(f'{self.name}:{self.marks}')
  • Objects are created from the class like this:
anish = Student('Anish', 90)

What is self?

Say, you want to call for example:

lakshmi.print_details()

Python invokes it as → Student.print_details(lakshmi)

self points to the current object (object attribute) whereas Student.count points to the class (class attribute)
  • self.name = name is the following operation: assign the value of the argument name to the current object's attribute!

Inheritance

Good example to consider is Questions, they come in variety of forms like MCQs, NAT, so on..

  • A parent class Question can be made to inherit:
class Question:
    def __init__(self, statement, marks):
        self.statement = statement
        self.marks = marks

    def print_question(self):
        print(self.statement)

    def update_marks(self, marks):
        self.marks = marks

Inherited in the following way:

class NAT(Question):
    def __init__(self, statement, marks, answer):
        super().__init__(statement, marks)
        self.answer = answer

    def update_answer(self, answer):
        self.answer = answer

Method Overriding:

Let’s say, if you want to print question with list of options, you have to define your own method that overrides parent class. For example:

class MCQ(Question):
    def __init__(self, statement, marks, ops, c_ops):
        super().__init__(statement, marks)
        self.ops = ops      # list of all options
        self.c_ops = c_ops  # list of correct options
    
    def print_question(self):
        super().print_question()
        # Assume there are only four options
        op_index = ['(a)', '(b)', '(c)', '(d)']
        for i in range(4):
            print(op_index[i], self.ops[i])
